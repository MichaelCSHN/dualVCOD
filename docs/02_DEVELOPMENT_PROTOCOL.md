# 开发纪律与代码提交流程

## 1. 里程碑驱动的 Commit
不要为每一行修改 Commit。只有当完成一个可运行的里程碑时才允许 Commit。
当前核准的里程碑路径：
*   [ ] M1: 跑通 VCOD 的多帧数据流和 BBox 评估器 (Dummy Test)。
*   [ ] M2: 实现 MicroVCOD 空间特征提取和时空邻域 (TN) 融合网络（独立 PyTorch 实现，非 GreenVCOD 复现）。
*   [ ] M3: 接入真实数据集（如 MoCA），跑通完整的前向、训练和 BBox 推理闭环。
*   [ ] M4: 记录单卡 4090 上的性能基准 (FPS, Params, BBox IoU)。

## 2. 每次修改的 CTO 质询
在编写任何代码前，请问自己：
1. 这个操作是否保持了视频处理的“轻量化”？会不会在 4090 上爆显存？
2. 这次修改是否让推理链条更干净、更易于最终部署为 BBox-only？
如果答案是否，请丢弃该想法。

## 3. 评估指标追踪 (Evaluation Priorities)
报告中必须包含以下数据的追踪：
1. `mean bbox IoU` 和 `Recall@0.5`（必须支持多帧平均）。
2. `FPS` (指定单卡 4090 测试，包含时序模块的前向时间)。
3. 模型参数量 `Params`。
